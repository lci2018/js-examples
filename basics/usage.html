Usage

Basics

The script tag

A JavaScript file can be added to your website using the script tag before the body's closing tag: 

 <script src="path/to/file.js"></script>
</body>

Script order

JS files are executed in order of appearance, therefore you should load external libraries (i.e. jQuery) before loading dependent files.

The console

The console can be found under the console tab in the browser's inspector. It allows us to debug our code. It can be used with the console.log() method to print messages and values.

Syntax

Writing

JavaScript is case sensitive; consistency is key when writing variables, functions, etc. Dashes are not tipically used to name variables or properties (since - is also the substraction symbol). Instead of using dashes, compound words are usually written in camel case (each word in the middle of the phrase begins with a capital letter). Example: variableName.

Quotation marks

Javascript uses quotation marks to enclose values that shouldn't be directly interpreted (usually text values). Double ("") or single ('') quotation marks can be equally used, but they should be used alternately if they are nested. Example:
"this is a 'quoted text'"

Brackets

JavaScript functions typically have one or more arguments (interpreted values) between brackets. The name of the function is written next to the brackets, without an space between them. Multiple arguments are separated by commas. Example: console.log("Hola");

Dots

Dots indicate belonging, usually to an object. For example, console.log() calls the log() function, which belongs to the console object.

Semicolons & curly brackets {}

Similarly to CSS, instructions typically end with a ; indicating that the next thing is a different instruction. Again similarly to CSS, several instructions can be grouped inside curly brackets {} and use them to link them to other elements, ie functions.

Comments in JS

Block comments are written with /* */, for example:
/* text inside a comment */

Inline comments are written with // at the start of the line, for example:
// Commented line
